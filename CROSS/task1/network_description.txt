Описание логики Задания 1 (Raw Sockets):

1. Инициализация:
   - Программа может работать в режиме сервера или клиента.
   - Используется AF_INET, SOCK_RAW, IPPROTO_UDP. Для raw-сокетов требуются права суперпользователя.

2. Клиент:
   - Формирует UDP-заголовок вручную.
   - Вычисляет контрольную сумму UDP, используя псевдозаголовок (IP отправителя, IP получателя, протокол, длина UDP).
   - При перехвате SIGINT/SIGTERM отправляет специальное сообщение SHUTDOWN_DISCONNECT.

3. Сервер:
   - Слушает все UDP пакеты на указанном порту.
   - Извлекает IP и UDP заголовки из полученного буфера (raw socket возвращает пакет целиком, включая IP header).
   - Хранит счетчики сообщений в связном списке (структура client_node_t), идентифицируя клиентов по паре IP:Port.
   - При получении SHUTDOWN_DISCONNECT удаляет клиента из списка, сбрасывая его счетчик.

4. Контрольная сумма:
   - Реализована по стандарту RFC 1071 (Internet Checksum).
